# P2 — Вики-база знаний ГП · спецификация (методика M2-Граф v2.1)

> Согласовано 2026-06-09. Развилки: JSON-граф + динамический рендер вики (FastAPI).
> Визуальное описание методики: `outputs/M2-methodology-v2.html`.

## Суть

Сшить две линии (entity-base 333 + operation-base) в единый типизированный граф связанных
объектов и отдать серфабельную вики. Процессная дисциплина — из метода П49.

## Конвейер A0→A6

A0 разведка (7 слоёв, 6 категорий, каркас-эталон) → A1 онтология графа → A2 план прохода →
A3 извлечение (операции + сущности → узлы + CI/VI + якоря + версии) → A4 агрегация в DAG →
**A4.5 сверка с каркасом** (diff каркас↔граф: пробелы/сироты) → A4.99 вердикт покрытия
(GO/CAUTION/LOOP/ABORT) → A5 противоречия (5 классов + same_as) → A6 синтез (вики, таймлайны, RAG-экспорт).
Петля качества: CI/VI; LOOP возвращает на дозаполнение.

## Модель графа

Узлы: Source · Fragment · Operation (S1–S6) · OperationType (A01–A18) · Entity (канон) ·
EntityVersion (инстанс во времени) · BackboneNode (каркас) · Author · Period.
Рёбра: produces · appears_in · cites · instantiates · transforms · revises (версии) ·
refines (между сущностями) · maps_to (entity→backbone) · precedes · same_as · relates_to.

## Совместимость с v1

Категории, 7 слоёв, S1–S6, entity.md, дельта-маркеры 🆕/🔄/✅/⚠️ сохраняются как атрибуты узлов.
Версии берутся из LOG.md и дельт. Каркас = курируемая «Структурная карта интегрированной онтологии».
Множество сущностей открытое: новые из новых источников, сверки с каркасом, дробного прохода, операций;
резолюция и сливает (same_as), и расщепляет.
